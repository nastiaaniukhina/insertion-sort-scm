# Лабораторна робота  
## Тема: Організація Software Configuration Management для навчального проєкту

---

## 🔹 Мета роботи
Навчитися застосовувати базові практики Software Configuration Management (SCM) у програмному проєкті: визначати об’єкти конфігурації, організовувати контроль змін, будувати traceability, фіксувати baseline і працювати з релізами.

---

## 🔹 Опис проєкту
Проєкт являє собою консольний застосунок на C++, який реалізує алгоритм сортування вставками (Insertion Sort) для масиву символів. Програма генерує випадковий масив, виконує сортування та виводить проміжні кроки.

# 📌 Контрольні питання

**1. Що таке SCM і чим він відрізняється від Git?**  
SCM — це система керування змінами всього продукту, Git — інструмент реалізації SCM.

**2. Що таке configuration item?**  
Будь-який елемент проєкту, який потрібно контролювати (код, конфігурації, документація).

**3. Що таке baseline?**  
Стабільний стан системи, який використовується як точка відновлення.

**4. Що таке traceability?**  
Можливість відстежити шлях зміни від ідеї до реалізації.

**5. Що таке release management?**  
Процес створення версій програмного продукту.

**6. Які об’єкти можуть бути під SCM?**  
Код, конфігурації, документація, CI/CD скрипти, середовище розгортання.
